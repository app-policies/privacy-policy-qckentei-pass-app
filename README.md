# QC検定Pass プライバシーポリシー

最終更新日: 2026年6月24日

QC検定Pass（以下「本アプリ」）は、品質管理検定（QC検定）2級・3級の学習機能、広告表示、アプリ内課金、利用状況の把握のために、端末上の学習履歴および外部サービスが提供する情報を利用する場合があります。

本アプリは、ログインやアカウント作成を必要としません。

## 収集する情報

- 学習履歴、回答履歴、正誤履歴、苦手問題、連続学習日数など、アプリ内で保存される学習データ
- アプリの設定情報
- 広告配信や広告効果測定に必要な情報
- アプリ内課金の購入状態を管理するために必要な情報
- 不具合調査、利用状況の把握、品質改善に必要な技術情報

本アプリは、氏名、住所、電話番号、メールアドレス、連絡先、正確な位置情報を、アカウント情報として独自に取得しません。

## 利用目的

- 問題演習、復習、学習進捗表示の提供
- 誤答優先学習、苦手問題、連続学習日数などの学習機能の提供
- 広告表示
- アプリ内課金の購入状態の確認、復元、管理
- 不具合調査、品質改善
- アプリ機能の利用状況を個人を特定しない範囲で把握するため

## 学習データの保存

学習履歴、回答履歴、苦手問題、連続学習日数などの学習データは、原則として利用者の端末内に保存されます。

本アプリは、学習履歴を管理するための独自アカウントサーバーを運営していません。端末の削除、アプリの削除、端末変更などにより、端末内の学習データが失われる場合があります。

## 広告について

本アプリでは、Google Mobile Ads を利用して、バナー広告、インタースティシャル広告、リワード広告を表示する場合があります。

リワード広告は、利用者が任意で視聴を選んだ場合に限り表示されます。広告視聴が正常に完了した場合、当日の学習で利用できる追加問題が解放されます。広告を読み込めない場合、表示に失敗した場合、または視聴が完了しなかった場合は、追加問題は解放されません。

Google Mobile Ads では、広告配信、広告効果測定、不正防止などのために、端末情報、広告配信に必要な識別子、利用状況に関する情報が利用される場合があります。これらの情報の取り扱いは、Google のポリシーに従って管理されます。

本アプリは、App Tracking Transparency の許可を求めるダイアログを表示せず、IDFA を利用しない方針です。

## アプリ内課金について

本アプリでは、非消耗型のアプリ内課金により、全問題の利用、広告非表示、詳細解説、選択肢ごとの解説、苦手問題モードなどの機能を提供する場合があります。

購入処理は Apple の仕組みを通じて行われ、購入状態の管理には RevenueCat を利用します。RevenueCat では、購入状態の確認、復元、課金機能の提供に必要な識別子や購入情報が取り扱われる場合があります。

## 利用状況の分析について

本アプリでは、Firebase Analytics を利用して、アプリの起動、学習開始、学習完了、模試完了、購入画面の表示など、機能改善のための利用状況を把握する場合があります。Firebase Analytics は広告識別子を利用しない設定で導入しています。

本アプリは、Firebase Authentication、Cloud Firestore、Cloud Functions、Cloud Storage、Remote Config、Crashlytics を利用しません。

利用状況の分析において、問題本文、選択肢本文、解説本文、自由入力の内容、氏名、メールアドレス、問題IDの生値、決済IDの生値、広告ID、各種SDKキーを送信しません。

## 第三者サービス

本アプリでは、広告配信、課金管理、品質改善のために、以下の第三者サービスを利用する場合があります。各サービスで取り扱われる情報は、それぞれのプライバシーポリシーに従って管理されます。

- Google Mobile Ads
- Firebase Analytics
- RevenueCat
- Apple が提供する App Store、アプリ内課金、クラッシュログ、診断関連サービス

## 外部送信

広告配信、課金管理、利用状況の分析、品質改善のため、第三者サービスに端末情報、購入情報、利用状況、診断情報などが送信される場合があります。

本アプリは、利用者が入力した氏名、住所、電話番号、メールアドレス、連絡先、正確な位置情報を独自に収集して外部送信する機能を持ちません。

また、問題本文、選択肢本文、解説本文、自由入力の内容、問題IDの生値、決済IDの生値、広告ID、各種SDKキーを外部送信しません。
